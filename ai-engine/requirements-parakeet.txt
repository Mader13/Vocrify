# ============================================================================
# PLATFORM-SPECIFIC NOTES
# ============================================================================
#
# Linux (Ubuntu 20.04+):
#   - Fully supported
#   - Install system dependencies:
#     sudo apt-get install -y ffmpeg libsox-dev libsox-fmt-all
#
# macOS:
#   - Supported with some limitations
#   - Install via Homebrew:
#     brew install ffmpeg sox
#
# Windows:
#   - LIMITED SUPPORT - Use WSL2 if possible
#   - Native Windows may have issues with nemo_toolkit
#   - If using native Windows:
#     1. Install ffmpeg: https://www.gyan.dev/ffmpeg/builds/
#     2. Add to PATH: setx PATH "%PATH%;C:\ffmpeg\bin"
#     3. May need to install Visual Studio Build Tools
#
# ============================================================================
# TROUBLESHOOTING
# ============================================================================
#
# Issue: ImportError: No module named 'nemo'
#   Solution: Reinstall nemo_toolkit
#     pip uninstall nemo-toolkit
#     pip install nemo-toolkit==2.2.1
#
# Issue: numpy version conflict
#   Solution: Keep numpy==1.24.4 (from base requirements.txt)
#     pip install numpy==1.24.4 --force-reinstall
#
# Issue: WebSocket error during inference
#   Solution: Upgrade websocket-client
#     pip install websocket-client --upgrade
#
# Issue: CUDA out of memory
#   Solution: Use CPU mode or reduce batch size in model config
#
